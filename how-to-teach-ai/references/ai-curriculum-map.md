# AI Tutorial Curriculum Map

Use this as a starting map. Adapt to the audience; do not force every chapter into every tutorial.

## Shared foundation

1. Why AI agents matter: from chat replies to delegated work.
2. What an AI agent is: goal, context, tools, actions, feedback loop.
3. Files and folders: what material the agent can see and change.
4. Command line without fear: why some tools need commands, what input/output means, how to cancel.
5. Accounts, network, and API keys: only after the learner understands the task that needs them.
6. Install/open one usable agent surface.
7. First safe task with disposable/sample files.
8. Read the agent interface: plan, trace, diffs, permissions, final answer.
9. Ask well: goal, constraints, evidence, stop condition.
10. Verify well: inspect outputs, run checks, compare source material.

## Office/admin branch

- Email drafting and formatting.
- Spreadsheet cleanup.
- Registration and activity data organization.
- Invoice/reimbursement material organization.
- Project progress summaries.
- Schedules, notices, and meeting follow-up.
- Word-style reports with validation-driven review.

## Research branch

- Literature extraction and notes.
- Data cleaning and plotting.
- Simulation/numerical scripts.
- Result sanity checks.
- Figure/report generation.
- Reproducible folder structures and run logs.

## Method branch

- Prompting as requirement writing.
- Plan → execute → verify loops.
- Validation-driven document generation.
- Working with examples and source files.
- Breaking a big task into safe small tasks.

## Safety branch

- Sensitive data and institutional boundaries.
- API keys and credentials.
- Permissions and sandbox prompts.
- Email/external-send approval.
- Human review before publication or decisions.
- Tool/model freshness and dated claims.
